Welcome to Automatic Data Scientist

Scala Machine Learning Toolbox

ADS – is “Machine Learning Studio” but with all extended functionalities to make data science process as easy and intuitive as possible.


Visualization – in other projects is usually considered as “side” functionality, but in ML world is extremely important – every time we create/update some model we need to properly present them. This is quite crucial task to visualize them and describe “why is so good”
Automatic Feature Engineering / data processing – ie: data normalization, transformations, extraction, - goal: to make input data distributed in way that makes ML works as close to 100% as possible
Algorithms discovery/ Algorithm selection – different types of data behaves differently - goal: use proper model for proper data Anomaly detection – in scenarios outside our domain, prediction is most desired functionality from ML, however specifics of our domain is usually to find and react to anomalies – goal: adjust “border parameters” to predict and identify anomaly – using automatic and manual inputs
Model interpretation /Model fitness – to interpret model/ describe and proof that model is working as expected ! (see LOCO/LIME algorithms) – tis is very hot trend in ML community now – since this is algorithm that automatically can tell that model build by data scientist is really working.
